GITHUB



Bienvenido a GitHub
Es un sistema de control de versiones, gratuito, de código abierto que permite el manejo de proyectos en colaboración de manera fácil, rápida y eficiente.





Paso 1: Crear un Repositorio
-Seleccionar New repository
-Nombrar el Repositorio e.j Prueba
-Seleccionar Initialize this repository with a README.
(El README contiene información sobre la funcionalidad del proyecto, es editable).
-Seleccionar Create new file para empezar con un archivo nuevo, o Upload files para subir un archivo a GitHub.

Repositorios: Son utilizados para la organización de proyectos. Estos pueden contener archivos, folder, multimedia, hojas de cálculo, etc.
Imaginelo como una archivero donde cada cajón es un repositorio que contiene varios tipos archivos.




Paso 2: Crear un Branch
-Ir al repositorio Prueba
-Click sobre el botón desplegable en la parte superior de la lista de archivos que dice branch: master.
-Escriba el nombre del branch e.j Edit dentro de la caja de texto.
-Dar "Enter" en el teclado para crear el nuevo branch.

Branching: También ramificación, consta en crear varias versiones del mismo código que son independientes del el branch master que es considerado la rama definitiva.
La ramificación se utiliza para experimentar y editar antes de fusionarlas con la rama master.
Se pueden crear una cantidad indefinida de brachs.




Paso 3: Crear Edits & Commits
-Seleccionar el archivo README.md
(Para fines del ejemplo se usara este archivo)
-Seleccionar el icono del lápiz en la parte superior derecha, para editar el archivo.
-Dentro del editor de texto escriba una "mini" biografía sobre usted.
-En la parte inferior, escriba un comentario, explicando sobre el cambio que realizo.
-Click sobre el botón Commit changes
Nota: Verifique que el commit que esta realizando sea sobre el branch Edit

Edist & Commits: Dentro de GitHub el “guardar cambios” se les llama commits.
Cada commit viene acompañado de un mensaje, en donde el se describe el cambio que se realizó y por qué lo hizo.




Paso 4: Open a Pull Request / Extraer una consulta
-Seleccionar el botón Pull request en la barra superior izquierda.
-Seleccionar el botón verde New pull request
-En la parte superios izquierda, seleccione en la caja compare: y seleccione el branch Edit para comparar con el branch "master", el cual es el original.
-Revise las comparaciones, el rojo indica el archivo original (branch master) y el verde indica los cambios que se realizaron dentro del branch Edit.
-Una vez satisfecho con los cambios realizados, seleccione el botón verde Create Pull Request para hacer una fusión entre el branch "master" (original) y el branch Edit
-Dentro de la caja de texto describa los cambios realizados.

Pull Request: Una vez que se hayan realizado los commits deseados, se crea un Pull request.
Es la revisión y comparación de el branch master y el brach nuevo que se haya creado.
Se presta para que otras personas revisen y comparen el código antes de realizar un merge.




Paso 5: Merge / Fusión
-Click sobre el botón verde Merge pull request para fusionar los cambios a "master".
-Click Confirm merge.
-Continúe y elimine la branch Edit, ya que sus cambios se han incorporado al branch "master", con el botón morado Delete.

Merge: Permite fusionar el o los branch al master una vez que se hayan concluidos los cambios deseados.
